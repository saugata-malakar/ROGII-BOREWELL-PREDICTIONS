What it does

A simple product story, not a code dump.

Reads well logs

It loads the well files, matches each horizontal well with its typewell companion, and creates a single consistent table per well.

Prepares usable features

It cleans missing values, creates depth and distance signals, and keeps only fields that the train and test sets truly share.

Returns a prediction file

It trains a model and writes the final TVT output in the exact submission format needed by Kaggle.

Problem detected

What was broken, and how the product fixes it.

Detected issue

Train and test did not match

The model initially saw different columns in training and testing data, which would normally break inference or create unusable predictions.

Detected issue

Data lived in multiple well files

Each well arrives as separate horizontal and typewell CSV files, so the app had to pair them correctly before the model could work.

Solved by the product

Feature alignment and safe inference

The pipeline keeps only shared fields, imputes missing values, engineers stable features, and produces a submission file in the competition format.
